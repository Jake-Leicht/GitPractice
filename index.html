Git Functions

  • Manage projects through repositories (repos)
  • Control/track changes with staging/committing
  • View/access past versions with branch/merge
  • Pull latest versions of a project
  • Push Changes on local repo to remote origin

File Changes

  • Changes to files are kept as untracked by default
  • To mark changes as tracked, the file must be staged
  • After staging the files you would like, you can commit those altered files
